Git 使用

  • git clone 克隆

    git clone 自动创建了一个名为 origin 的远端连接，指向所 fork 的仓库。

  • git checkout -b dev 创建新分支

  • git push -u origin dev 直接创建远程 dev 分支

  • git branch 查看分支

  • git branch -D dev 删除本地 dev 分支

  • git branch -delete dev 删除本地 dev 分支

  • git branch -r -D origin/dev 删除本地远程 dev 分支

  • git push origin -d dev 删除远程git服务器的分支

  • git checkout dev 切换 dev 分支

git 提交代码

git status

git diff

git add .

git commit -m ''

git pull origin dev

git push origin dev

  • git remote 添加远程

git remote add origin <url>

git remote -v

git remote set-utl <url>

git remote add <name> <url>



git remote -v

// 打印下面的结果

origin  https://github.com/xx/xxx.git (fetch)

origin  https://github.com/xx/xxx.git (push)

upstream    https://github.com/xx/xxx.git (fetch)

upstream    https://github.com/xx/xxx.git (push)



// 从 original repo 获取更新，并merge到本地

git fetch upstream

git merge upstream/70_dev

// 下面的指令 short for 上面的2个指令

git pull upstream 70_dev

  • git merge 合并分支

git checkout dev

git merge master



git checkout dev

git rebase master

git rebase 指令冲突以后，编辑冲突的文件，然后执行：
git add -u (-u表示把所有已track的文件的新的修改加入缓存，但不加入新的文件)
git rebase –continue （反复执行这两行指令，直到所有的冲突解决掉。）

获取远程指定分支：
git clone -b name url
当前仓库太大时，就加上 –depth=数字。获取指定深度。
git clone –depth=6 -b name url

有时候 git ignore不起作用，有可能是配置的ignore文件中忽略的文件已经被添加到git版本管理中。此时就需要清理git tracked的缓存文件。方法是：
git rm -r –cached . (注意后面有个点，这命令作用是将所有文件都取消跟踪)
git add .
git commit -m ‘update git tracked files’
